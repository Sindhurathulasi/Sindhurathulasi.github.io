Leadsquared UI test

Task 1

A website asks you for a phone number and needs you to get it right. When you enter your ten-digit phone number, you see something like this. Please suggested improve UI for this phone control.

Solution: The key to good user experience is preventing the user from making an error than correcting them after.
- Adding the flag icon aids preventing user from having to type the country code extension
- Auto-formatting with respect to the country code prevents the user from inputting a wrong format
- Masking the input makes it easier for the users to validate the number

Task 2

How can you improve this feedback form?

Solution: In any feedback form, asking the right questions is the key. The users don’t really have the patience to completely read and fill the form.
- Framed the question differently so that it can be read easily
- As the level of satisfaction is number based, I put them in an input selector instead of hiding the options in a dropdown
- Added the parameters (Not very satisfied - Very satisfied) close to the number scale range (0 - 10) so that the user’s eye need not travel back and forth for reference
- Highlighted the required field as it is only one field as opposed to the optional fields

Task 3

Redesign this parking sign which can drive anyone crazy. When you are trying to park a car and see the sign, you have very limited time to understand the sign. So your design should consider that aspect – which is an individual should be able to understand at a single glance whether they can park at that moment or not. The design should also consider the fact that the number of constraints can increase over a period of time – meaning the sign below has constraints on Friday through Monday. There could be additional constraints on other weekdays and hours in the future.

Solution:
- Arrow being on top makes it easier to understand the direction the parking guide
- As there can be additional constraints on the other weekdays, I have added all days and given them space so that those changes can be accommodated without any issue
- Used graphical icons to represent different signs as it makes it easier for people to scan through the sign board and understand if they can park or not
- Added differentiation for paid and free parking to make it more clear
- Added legend so that it will be easier for all kinds of users ranging from different age groups to understand better and faster
- Color is another factor that aids in scanning the sign board easily, used green and red for the do's and don’t respectively
- Font used is Helvetica as it is very easily readable and simple as that is an important factor for legibility
- As it also contains paid parking, it is important to have a way to be able to easily pay or atleast review the fares. That is the reason for adding the QR code option as another way to pay
- Tow-away is another use case scenario where it is important to help users understand who to reach out to to get any information with regard to their towed vehicle

Task 4

Design a better 500 internal server error page

Solution: Whenever there is an occurrence of an error, users generally tend to get lost, so it is very important to give them help and redirect them to the right page.
- Added ways for the user to reach out and also linked the help center for any additional information
- Option to redirect to a relevant page
- Error messages can be annoying to the users, so adding good illustration will always aid in the look and feel of the page
- The way the error is communicated is also important, it is always good to inform the user that the error is being worked on
